# Jyotish Strategy

Last updated: 2026-04-09

## Purpose

Define how Sanatana Sangam can introduce Jyotish / Rashi guidance without collapsing into a generic horoscope product.

## Product posture

Jyotish should be framed as:

- reflective guidance
- opt-in
- dharmic orientation
- clearly limited in scope

It should not be framed as:

- certainty
- prediction-as-entertainment
- universal truth for every user without source clarity

## Launch order

1. Festival truth layer
2. Panchang trust and source clarity
3. Jyotish definition
4. Later Rashi guidance

## First allowed version

The first production-safe version should be:

- `Rashi insights` or `Jyotish guidance`
- transit-based and clearly labeled
- daily or weekly reflective guidance
- tied to sacred-time rhythm, not sensational copy

## Not for the first version

- high-confidence personal prediction
- vague mass-market horoscope language
- birth-chart claims without reliable birth data and method disclosure

## Later expansion path

If trust stays strong, later versions can add:

- birth date
- birth time
- birth place
- janma rashi / nakshatra based guidance

But only after:

- data privacy is clear
- method choice is defined
- UX language stays humble and transparent

## Relation to Pathshala and sacred time

Jyotish should sit under one `Sacred Time` umbrella with:

- Panchang
- festivals
- later Rashi guidance

This keeps it aligned with practice and rhythm instead of making it feel like a separate horoscope app.
